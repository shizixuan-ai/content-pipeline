## 腰部 2：核心能力拆解（任务：展示功能价值）

### 模块 1：七阶段创作流水线
- drafting → outlining → writing → reviewing → polishing → finalized → distributing
- 中断恢复 + 用户确认流转
- 每阶段独立 prompt，角色隔离

### 模块 2：平台人格化改写
- 每个平台配置 Persona（受众、语气、禁忌）
- 掘金=技术深度 / 知乎=论点论据 / 简书=故事化 / 公众号=排版 / CSDN=实战 / 头条=热点
- 自动格式降级（不支持的特性自动 fallback）

### 模块 3：API 一键分发
- Cookie/OAuth2/Playwright 三种认证模式
- 6 平台统一接口: `distribute.sh <article> <platform...>`
- 幂等保护 + 探活验证 + 失败隔离
- {金句：6 个平台，5 分钟，一条命令}
- [建议：插入终端录屏 demo GIF]

## 腹部：对比与差异化（任务：说清为什么不用竞品）

| 维度 | 手工分发 | 其他工具 | content-pipeline |
|------|---------|---------|------------------|
| 平台覆盖 | 靠手 | 单一平台 | 6 平台(掘金/公众号/头条/知乎/简书/CSDN) |
| 人格化改写 | 自己改 | 无 | 按平台 Persona 自动改写 |
| 幂等保护 | 无 | 无 | MD5 校验 + 分发日志 |
| 探活验证 | 无 | 无 | HTTP HEAD 自动验证 |
| 失败隔离 | 全挂 | 全挂 | 单平台失败不影响其他 |
| 认证方式 | Cookie 手动 | 单一 | Cookie/OAuth2/Playwright 三重 |

{金句：不是工具不够多，是没有专门为内容管线设计的工具}
