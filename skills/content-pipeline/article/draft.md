## content-pipeline 是什么

一句话：**一次写作，按平台改写，API 一键分发。**

它是一个开源的内容管线工具。你的全部写作流程 — 从定题、写大纲、出初稿、审查修改、排版美化，到最终分发到 6 个平台 — **全部在这一个管线里完成**。

<!-- image: flowchart | pipeline 七阶段流程 drafting→outlining→writing→reviewing→polishing→finalized→distributing | 让读者一眼看懂完整创作流水线 -->

不只是"发文章"这么简单。每个阶段都隔离成独立角色：

| 阶段 | 角色 | 产出 |
|------|------|------|
| Drafting | 需求分析师 | 文章要点 brief.md |
| Outlining | 内容架构师 | 有条理的大纲 outline.md |
| Writing | 技术写作专家 | 完整初稿 draft.md |
| Reviewing | 毒舌主编 | 审查报告 |
| Polishing | 排版师 | WeChat 兼容 HTML |
| Distributing | 分发工程师 | 6 平台一键投递 |

每个阶段完成后，你看一眼，点头，再进下一个。**中断了也不怕** — 下次进来直接从断点续。

## 核心能力拆解

### 能力 1：平台人格化改写

每个平台的读者不一样。你在掘金讲技术深度，在知乎需要论点论据，在简书得讲故事。

content-pipeline 为每个平台配了一份 **Persona 档案**：

```yaml
# platforms/zhihu.md
audience: 大学生、高知群体
mindset: 喜欢看"为什么"
pain: 缺乏论据的观点、无数据支持的断言
taboo:
  - 泛泛而谈
  - 无数据支持的结论
```

分发时，管线按 Persona **自动改写语气和结构**。掘金版保留完整代码，简书版把代码打成故事片段，知乎版加上对比表格和前沿数据引用。

不需要你自己记 —— 每个平台要什么风格，**管线替你记住了**。

<!-- image: diagram | 同一篇文章在 6 平台的不同呈现 | 展示人格化改写的效果 -->

### 能力 2：格式智能降级

不是所有平台都支持 Mermaid 图和数学公式。知乎支持 `$$`，简书不支持。

管线读每个平台的 `Format Capabilities` 配置，**自动降级不支持的格式**：

```
markdown_features:
  mermaid: false     # → 降级为截图
  math: false        # → 降级为文字描述
  footnotes: false   # → 降级为行内注释
```

你只管写。兼容的事交给 pipeline。

### 能力 3：一键分发 + 幂等保护

这是最直接的爽点。

```bash
./scripts/distribute.sh ./article juejin zhihu csdn jianshu
```

**一条命令，5 分钟，6 个平台全部发完。**

<!-- image: user-supply | 终端录屏 demo GIF | 展示命令行分发全过程 -->

底层做了三件事：

1. **认证链** — Cookie / OAuth2 / Playwright 三种模式，每个平台自动认证
2. **失败隔离** — 知乎挂了，不影响掘金和 CSDN 正常发布
3. **幂等保护** — 已分发的文章 MD5 校验，不小心重跑也会**警告你**，不会重复发

{金句：6 个平台，5 分钟，一条命令}

## 对比：手工分发 vs Pipeline

| 维度 | 手工分发 | 其他工具 | content-pipeline |
|------|---------|---------|------------------|
| 平台覆盖 | 逐个手动粘贴 | 单一平台 | 掘金 / 公众号 / 头条 / 知乎 / 简书 / CSDN |
| 人格化改写 | 自己记、自己改 | 无 | 按 Persona 自动改写 |
| 防重复发布 | 无 | 无 | MD5 校验 + 分发日志 |
| 发布验证 | 发完不管 | 无 | HTTP 探活自动确认 |
| 容错 | 一个平台卡住全停 | 全挂 | 单平台失败不影响其他 |
| 认证方式 | 手动复制 Cookie | 单一认证 | Cookie + OAuth2 + Playwright |

**不是工具不够多，是没有专门为内容管线设计的工具。**

## 效果：省了多少

**手工分发 6 个平台：约 1.5 小时。用 pipeline：约 5 分钟。**

省了 **95% 的时间**。一个月发 4 篇，一个月省 6 小时。一年省出 **3 个工作日**。

这些时间你本来可以用于：写更好的文章、读更多的文档、陪家人。或者，就发个呆也挺好。

### 上手三步

```
第一步：克隆项目
git clone && npm install

第二步：配置认证
./scripts/login-juejin.sh    # 扫码一次，永久有效
export CSDN_COOKIE="..."     # 或从浏览器复制

第三步：写作 + 分发
# 从写大纲到发布，全在管线里
./scripts/distribute.sh ./article juejin zhihu csdn jianshu
```

**一次配置之后，第二条、第三条文章就只是「写」和「发」两步了。**
