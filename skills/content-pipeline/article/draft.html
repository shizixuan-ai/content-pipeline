content-pipeline 是什么

一句话：一次写作，按平台改写，API 一键分发。

它是一个开源的内容管线工具。你的全部写作流程 — 从定题、写大纲、出初稿、审查修改、排版美化，到最终分发到 6 个平台 — 全部在这一个管线里完成。

每个阶段完成后，你看一眼，点头，再进下一个。中断了也不怕 — 下次进来直接从断点续。

核心能力拆解

能力 1：平台人格化改写

每个平台的读者不一样。你在掘金讲技术深度，在知乎需要论点论据，在简书得讲故事。

content-pipeline 为每个平台配了一份 Persona 档案：

# platforms/zhihu.md
audience: 大学生、高知群体
mindset: 喜欢看"为什么"
pain: 缺乏论据的观点、无数据支持的断言
taboo:
  - 泛泛而谈
  - 无数据支持的结论

分发时，管线按 Persona 自动改写语气和结构。掘金版保留完整代码，简书版把代码打成故事片段，知乎版加上对比表格和前沿数据引用。

不需要你自己记 — 每个平台要什么风格，管线替你记住了。

建议作者在此处插入：对比图——展示人格化改写的效果

能力 2：格式智能降级

不是所有平台都支持 Mermaid 图和数学公式。知乎支持 $$，简书不支持。

管线读每个平台的 Format Capabilities 配置，自动降级不支持的格式：

markdown_features:
  mermaid: false     # → 降级为截图
  math: false        # → 降级为文字描述
  footnotes: false   # → 降级为行内注释

你只管写。兼容的事交给 pipeline。

能力 3：一键分发 + 幂等保护

这是最直接的爽点。

./scripts/distribute.sh ./article juejin zhihu csdn jianshu

一条命令，5 分钟，6 个平台全部发完。

建议作者在此处插入：终端录屏 GIF——展示命令行分发全过程

底层做了三件事：

  1. 认证链 — Cookie / OAuth2 / Playwright 三种模式，每个平台自动认证
  2. 失败隔离 — 知乎挂了，不影响掘金和 CSDN 正常发布
  3. 幂等保护 — 已分发的文章 MD5 校验，重跑会警告你，不会重复发

金句：6 个平台，5 分钟，一条命令

效果：省了多少

手工分发 6 个平台：约 1.5 小时。用 pipeline：约 5 分钟。

省了 95% 的时间。一个月发 4 篇，一个月省 6 小时。一年省出 3 个工作日。

这些时间你本来可以用于：写更好的文章、读更多的文档、陪家人。或者，就发个呆也挺好。

上手三步

# 第一步：克隆项目
git clone && npm install

# 第二步：配置认证
./scripts/login-juejin.sh    # 扫码一次，永久有效
export CSDN_COOKIE="..."     # 或从浏览器复制

# 第三步：写作 + 分发
./scripts/distribute.sh ./article juejin zhihu csdn jianshu

一次配置之后，第二条、第三条文章就只是「写」和「发」两步了。
