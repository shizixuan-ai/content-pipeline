# 审核清单 (CHECKLIST)

> 此清单用于 Reviewing 阶段逐项检查文章质量。

## 结构检查

- [ ] 文章有清晰的结构，读者一看就知道在讲什么
- [ ] 开头 3 秒能抓住注意力（痛点/反直觉结论/场景）
- [ ] 每个段落都有明确的任务属性（引出焦虑、提供证据、升华价值等）
- [ ] 结尾有明确的行动指令（不是戛然而止）
- [ ] 没有"烂尾"：尾部不敷衍，有互动引导

## 内容检查

- [ ] 每段都有"金句"或核心观点，不是流水账
- [ ] 有具体的数字/案例/代码，不只是空谈
- [ ] 没有"正确的废话"（如"AI 正在改变世界"这种谁都知道的话）
- [ ] 每个"建议"都有"为什么"的支撑，不强行推销

## 语言检查

- [ ] 像人话：读起来像是一个人在说话，不是在念报告
- [ ] 没有 AI 黑话词汇（对照 blacklist-words.md 逐项检查）
- [ ] 没有"AI 味连接词"（综上所述、总而言之、通过以上分析）
- [ ] 每句话不超过 30 字（超长句必须拆分）
- [ ] 段落开头句式不重复（不要每段都以"首先/其次/最后"或"在...背景下"开头）

## 排版检查

- [ ] 每段不超过 3 行（手机屏幕约 5-6 行）
- [ ] 每段都有粗体锚点（4-7 字）
- [ ] 引用块用于补充说明，不打断主线
- [ ] 代码块前后有留白
- [ ] 大章节之间有空行或分隔线

## 违规检查（硬性规定）

- [ ] 无辱骂/人身攻击词汇
- [ ] 无政治敏感内容
- [ ] 无对立挑动（地域/性别/职业）
- [ ] 无违规引流/诱导分享
