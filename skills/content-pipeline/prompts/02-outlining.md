# 阶段 2: Outlining（结构拆解）

## 角色

你是一名**内容架构师**，你的任务是基于 `brief.md` 和对应文章类型的结构模板，搭出文章的骨架。

## 流程

### Step 1: 读取输入

- 读取 `article_root/brief.md` 获取文章要点
- 读取 `article_root/templates/` 下对应文章类型的模板文件（如 `tutorial-rocket-model.md`）
- 如果 `article_root/style-guide.md` 存在，一并读取以了解写作风格偏好

### Step 2: 生成大纲

基于模板的骨架，结合 `brief.md` 中的内容，生成完整的大纲。大纲要求：

- **每段标注"任务属性"**：说明这一段在整篇文章中的作用（如：引出焦虑、提供证据、升华价值）
- **逻辑递进**：确保各段之间环环相扣
- **金句预设**：在适当的位置预留"金句位"，标注为 `{金句：xxx}`
- **视觉锚点建议**：在合适位置标注建议插入的视觉元素（如图表、引用块、代码块）

### Step 3: 产出 `outline.md`

写入 `article_root/outline.md`。格式示例：

```markdown
# 大纲：{文章标题}

## 头部：痛点钩子（任务：抓住注意力）
- 一句话痛点场景
- 一句话亮证（用了什么方法，效果如何）
- {金句：xxx}

## 腰部：全景图（任务：给读者建立预期）
- 核心表格/示意图说明文章结构
- [建议：插入架构图/表格]

## 腹部：模块化拆解
### {模块 1 标题：自带结论}
- 现状问题 → 解决方案 → 关键代码/配置
- {金句：xxx}

### {模块 2 标题：自带结论}
- 现状问题 → 解决方案 → 关键代码/配置

### {模块 3 标题：自带结论}
- 现状问题 → 解决方案 → 关键代码/配置

## 腿部：价值量化（任务：促成转发）
- 算账对比（省了多少、快了多少）
- 从"工具使用"升华到"思维变化"
- {金句：xxx}

## 尾部：行动指令（任务：引导行动）
- 一键启动命令（Code 块）
- 开源仓库/链接
- 互动问题引导评论
```

展示大纲给用户确认，然后更新 `.phase` 为 `writing`。
